# 台本生成スキル

動画教材・プレゼン・講義用の台本Markdown（script.md）を生成する。
構成案の確認 → ライティング の2段階で進める。

## ワークフロー

### Step 1: インプットを聞く

ユーザーに以下を確認する：
- テーマ・タイトル
- 対象時間（例: 30分、1時間）
- ターゲット（例: プログラミング未経験者、エンジニア向けなど）
- 含めたい内容・こだわりポイント（あれば）
- トーン・話し方の希望（あれば。例: 敬語でやわらかく、など）

すでにユーザーが情報を提供済みの場合はスキップしてよい。

### Step 2: 構成案を提示する

台本の構成案を箇条書きの目次形式で提示する。

**フォーマット：**

```
# タイトル

## 概要
- 対象時間：XX分
- このコンテンツで学ぶこと
  - 学習項目1
  - 学習項目2
  - ...

## 台本構成

### [セクション名1]
- トピックA：簡単な説明
- トピックB：簡単な説明

### [セクション名2]
- トピックC：簡単な説明
- トピックD：簡単な説明

### [まとめ]
- 振り返りポイント
- 次へのつながり
```

**構成案のルール：**
- セクションは `### [セクション名]` の形式で列挙する
- 各セクション内のトピックは箇条書きで、1行に概要を添える
- トピックの粒度は「1つの話題 = 1項目」
- 導入（つかみ）とまとめを必ず含める

### Step 3: ユーザー確認（必須）

構成案を提示したら、必ずユーザーに確認を求める。

```
この構成でよろしいですか？
追加・削除・順番の変更があれば教えてください。
```

ユーザーから「OK」「いいね」「go」等の承認が得られるまでライティングに進まない。
修正指示があった場合は構成案を修正して再提示する。

Step 2〜3 はユーザーがOKを出すまで何度でもループする。

### Step 4: ライティング

ユーザーの承認を得たら、構成案に基づいて台本本文を書く。

**ライティングのルール：**

- 話し言葉で書く。読み上げたときに自然に聞こえること
- 一文は40文字以内を目安
- `**太字**` は絶対に使わない
- 専門用語は初出時に必ず簡単な説明を添える
- ト書き（画面操作の指示など）は `【操作】〜する` の形式
- 間を置きたい箇所は `（少し間）` と書く
- 絵文字は使わない
- 具体例を豊富に入れる

**文字数の目安：**
- 30分の動画 → 10,000文字以上
- 1時間の動画 → 20,000文字以上
- 1.5〜2時間の動画 → 25,000〜35,000文字以上

**出力ファイル：**
- ユーザーが指定した場所、または現在のディレクトリに `script.md` として保存する

### Step 5: 文字数を表示する

台本完成後、文字数を表示する。

```bash
wc -m script.md
```

## 注意事項

- 構成案の確認を飛ばしてライティングに入らないこと（Step 3は必須）
- 台本の内容をリライトや要約せず、話し言葉のままフルで書くこと
